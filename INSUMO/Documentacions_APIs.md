📚 Referencia Maestra de APIs y Fuentes de Datos (OAR Prototipo)Este documento sirve como guía para Antigravity (y desarrolladores futuros) sobre las fuentes oficiales, documentación de APIs y estrategias de datos utilizadas en los nuevos reportes técnicos del OAR.1. 🌲 Bosques y Cobertura (GFWReport.jsx)Este componente se alimenta actualmente de datos pre-procesados para evitar cargas masivas en el cliente, pero la fuente original es Global Forest Watch.Componente: src/pages/technical/GFWReport.jsxFuente de Datos Actual: src/api/sicaDataProcessed.js (Generado vía Python desde CSVs oficiales).Fuente Oficial: Global Forest Watch (WRI).Portal de Datos: data.globalforestwatch.orgAPI Docs: GFW Developer PortalDatasets Clave: * Tree Cover Loss (Hansen/UMD/Google/USGS/NASA).Forest Carbon Flux (Harris et al.).2. 🛡️ Biodiversidad - Meta 30x30 (BioReport.jsx)Monitor de progreso hacia la Meta 3 del Marco Global de Biodiversidad Kunming-Montreal.Componente: src/pages/technical/BioReport.jsxFuente Oficial: UN Biodiversity Lab (UNBL) & CBD.Plataforma: unbiodiversitylab.orgReferencia Técnica: UNBL Data CollectionsIndicadores: * Protected Area Coverage (WDPA).Key Biodiversity Areas (KBAs).Species Threat Abatement and Restoration (STAR) metric.3. 🐾 Biodiversidad en Tiempo Real (GBIFDashboard.jsx)Conexión viva a la infraestructura mundial de información en biodiversidad.Componente: src/pages/technical/GBIFDashboard.jsxFuente Oficial: GBIF (Global Biodiversity Information Facility).API Docs: GBIF API ReferenceEndpoints Utilizados:Búsqueda: https://api.gbif.org/v1/occurrence/search (Filtros: country, hasCoordinate, mediaType).Mapas (Calor): https://api.gbif.org/v2/map/occurrence/density/{z}/{x}/{y}@1x.pngNotas: API abierta, no requiere Key para métodos GET públicos.4. 🏞️ Áreas Protegidas (ProtectedPlanetDashboard.jsx)La base de datos oficial de la ONU para áreas protegidas.Componente: src/pages/technical/ProtectedPlanetDashboard.jsxFuente Oficial: Protected Planet (UNEP-WCMC).API Docs: Protected Planet APIEndpoints: https://api.protectedplanet.net/v3/countries/{iso3}Notas: Requiere Token gratuito para producción. El prototipo usa un modo "Fallback" con datos cacheados si no hay token presente.5. 🔥 Monitor de Incendios (FiresDashboard.jsx)Sistema de información sobre incendios para la gestión de recursos.Componente: src/pages/technical/FiresDashboard.jsxFuente Oficial: NASA FIRMS (Fire Information for Resource Management System).API Docs: NASA FIRMS APISensores: VIIRS (375m) y MODIS (1km).Endpoint Típico: /api/country/csv/{MAP_KEY}/VIIRS_SNPP_NRT/{ISO_CODE}/{DAYS}Notas: Los datos se actualizan casi en tiempo real (NRT - Near Real Time).6. 🌦️ Clima y Corredor Seco (ClimateDashboard.jsx)Datos meteorológicos históricos y pronósticos para agricultura.Componente: src/pages/technical/ClimateDashboard.jsxFuente Oficial: Open-Meteo (Datos abiertos no comerciales).API Docs: Open-Meteo DocsEndpoint: https://api.open-meteo.com/v1/forecastParámetros: latitude, longitude, daily=temperature_2m_max,precipitation_sum.Notas: Excelente para series temporales sin necesidad de API Key 
compleja.7. 💧 Seguridad Hídrica (WaterDashboard.jsx)Atlas de riesgos hídricos del WRI.Componente: src/pages/technical/WaterDashboard.jsxFuente Oficial: WRI Aqueduct Water Risk Atlas.Plataforma: wri.org/aqueductDatos: Aqueduct 4.0 Data.Notas: Aqueduct no tiene una API JSON simple para consultas en vivo frontend; se recomienda usar sus capas vectoriales o datos tabulares pre-procesados (como hace el componente actual).8. 🌊 Océanos y Corales (OceanDashboard.jsx)Monitoreo de blanqueamiento de corales y temperatura superficial.Componente: src/pages/technical/OceanDashboard.jsxFuente Oficial: NOAA Coral Reef Watch (CRW).Plataforma: coralreefwatch.noaa.govProductos: Regional Virtual Stations (Series temporales de 5km).Formatos: JSON, NetCDF, imágenes GeoTIFF.
